明白，你想在 **C++ / CMake 项目中管理多环境的 `.env` 文件**，比如 **开发（dev）、测试（test）、生产（prod）**，并且能在不同环境下自动切换。下面是一些常用做法和建议。

---

## 1️⃣ 使用多 `.env` 文件

在项目根目录创建多个环境文件，例如：

```
.env.dev
.env.test
.env.prod
```

内容示例：

```env
# .env.dev
REDIS_PLUS_PLUS_INCLUDE_DIR=D:/dev/vcpkg/installed/x64-windows/include
REDIS_PLUS_PLUS_LIBRARY=D:/dev/vcpkg/installed/x64-windows/lib/redis++.lib
HIREDIS_LIBRARY=D:/dev/vcpkg/installed/x64-windows/lib/hiredis.lib
NPCAP_INCLUDE_DIR=D:/npcap-sdk-1.15/Include
NPCAP_LIBRARY=D:/npcap-sdk-1.15/Lib/x64/wpcap.lib
```

```env
# .env.prod
REDIS_PLUS_PLUS_INCLUDE_DIR=E:/prod/vcpkg/installed/x64-windows/include
REDIS_PLUS_PLUS_LIBRARY=E:/prod/vcpkg/installed/x64-windows/lib/redis++.lib
HIREDIS_LIBRARY=E:/prod/vcpkg/installed/x64-windows/lib/hiredis.lib
NPCAP_INCLUDE_DIR=E:/npcap-sdk-1.15/Include
NPCAP_LIBRARY=E:/npcap-sdk-1.15/Lib/x64/wpcap.lib
```

## 2️⃣ 在 CMake 中选择 `.env` 文件

可以通过 **CMake 变量**来指定环境：

```powershell
# 命令行传入
cmake -S . -B build -DENVIRONMENT=dev
```

CMakeLists.txt：

```cmake
# 默认环境
if(NOT DEFINED ENVIRONMENT)
    set(ENVIRONMENT "dev")
endif()

# 构建 .env 文件路径
set(ENV_FILE "${CMAKE_SOURCE_DIR}/.env.${ENVIRONMENT}")

if(EXISTS "${ENV_FILE}")
    file(READ "${ENV_FILE}" ENV_CONTENTS)
    foreach(line IN LISTS ENV_CONTENTS)
        if(line MATCHES "^\\s*([^=]+)=(.+)$")
            set(${CMAKE_MATCH_1} "${CMAKE_MATCH_2}")
        endif()
    endforeach()
    message(STATUS "Loaded environment: ${ENVIRONMENT}")
else()
    message(WARNING "Environment file ${ENV_FILE} not found")
endif()
```

---

## 3️⃣ 在 CI 或脚本中切换环境

例如 GitHub Actions 或 PowerShell：

```powershell
# 选择环境
$env:ENVIRONMENT = "test"

# 调用 CMake
cmake -S . -B build -DENVIRONMENT=$env:ENVIRONMENT
```

这样就会加载对应的 `.env.test` 文件。

---

## 4️⃣ 优点

1. **每个环境独立**，路径和配置不会混乱
2. **可版本控制** `.env.*` 文件，团队共享一致
3. **可扩展**：增加数据库、SDK 或第三方库路径只需要修改对应环境文件
4. **CI/CD 兼容**：只需在构建脚本里切换 `ENVIRONMENT` 即可

---

💡 **建议实践**：

* 保留 `.env.dev` 在仓库里，其他环境（prod、staging）可放在私有仓库或 CI secret
* 避免把敏感信息直接写在 `.env` 文件中，可通过 CI secret 覆盖
