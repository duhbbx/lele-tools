# OpenCV 图像处理模块

这是一个基于 OpenCV 的综合图像处理工具，集成了丰富的图像处理功能。

## 功能特性

### 🖼️ 图像输入/输出
- **图像读取**: 支持 PNG、JPG、JPEG、BMP、TIFF、WebP、GIF 等常见格式
- **图像保存**: 可保存为多种格式
- **剪贴板操作**: 支持复制图像到剪贴板和从剪贴板粘贴
- **视频处理**: 支持打开视频文件进行帧处理
- **摄像头支持**: 实时摄像头图像处理

### 🎨 基础图像处理
- **灰度化**: 将彩色图像转换为灰度图像
- **二值化**: 固定阈值二值化处理
- **自适应阈值**: 自适应阈值二值化，适应不同光照条件
- **高斯模糊**: 使用高斯核进行模糊处理
- **中值滤波**: 去除椒盐噪声
- **双边滤波**: 保边去噪滤波
- **锐化**: 增强图像边缘和细节
- **直方图均衡化**: 增强图像对比度
- **CLAHE**: 限制对比度自适应直方图均衡化

### 🔍 边缘检测
- **Canny 边缘检测**: 经典边缘检测算法，可调节高低阈值
- **Sobel 边缘检测**: 基于梯度的边缘检测
- **Laplacian 边缘检测**: 基于二阶导数的边缘检测

### 📍 角点检测
- **Harris 角点检测**: 检测图像中的角点特征（红色圆圈标记）
- **Shi-Tomasi 角点检测**: Good Features to Track 角点检测（绿色圆圈标记）

### ⚡ 几何变换
- **缩放**: 自定义 X 和 Y 方向缩放比例
- **旋转**: 绕图像中心旋转指定角度
- **仿射变换**: 三点对应的仿射变换
- **透视变换**: 四点对应的透视变换

### 🔧 滤波器
- **均值滤波**: 简单均值模糊
- **高斯滤波**: 高斯核滤波
- **Sobel 滤波**: Sobel 算子滤波
- **Laplacian 滤波**: Laplacian 算子滤波

## 使用方法

### 基本操作流程
1. **打开图像**: 点击"打开图片"按钮选择图像文件
2. **选择处理模式**: 在处理模式下拉框中选择需要的处理类型
3. **调整参数**: 在参数设置标签页中调整相应参数
4. **应用处理**: 点击"应用处理"按钮执行图像处理
5. **保存结果**: 使用"保存图片"或"复制"按钮保存处理结果

### 界面布局
- **左侧**: 图像显示区域，支持缩放和拖拽
- **右侧**: 控制面板，包含三个标签页：
  - **处理模式**: 选择处理算法和执行处理
  - **参数设置**: 调整算法参数
  - **使用说明**: 查看功能介绍

### 图像显示操作
- **鼠标滚轮**: 缩放图像
- **左键拖拽**: 平移图像
- **自动适应**: 图像加载时自动适应窗口大小

### 视频/摄像头模式
- **打开视频**: 选择视频文件进行逐帧处理
- **打开摄像头**: 实时处理摄像头图像
- **实时处理**: 在视频模式下可实时应用某些处理效果

## 参数说明

### 二值化参数
- **阈值**: 二值化的分割阈值 (0-255)
- **最大值**: 高于阈值的像素值 (0-255)

### Canny 边缘检测参数
- **低阈值**: 边缘连接的低阈值 (0-255)
- **高阈值**: 强边缘的高阈值 (0-255)

### 模糊处理参数
- **核大小**: 滤波核的大小，必须为奇数 (1-31)

### 几何变换参数
- **缩放**: X 和 Y 方向的缩放比例 (0.1-5.0)
- **旋转**: 旋转角度，正值顺时针，负值逆时针 (-180° 到 180°)

## 技术特点

### 多线程处理
- 图像处理在独立工作线程中执行
- 避免界面卡顿，提供处理进度反馈
- 支持处理过程中的错误处理和恢复

### 内存管理
- OpenCV Mat 和 QImage 之间的高效转换
- 自动内存管理，避免内存泄漏
- 大图像的优化处理

### 错误处理
- 完善的异常处理机制
- 用户友好的错误提示
- 处理失败时的状态恢复

## 编译要求

- **OpenCV 4.x**: 需要启用 WITH_OPENCV 宏
- **Qt 6.x**: 使用 Qt 的 GUI 框架
- **C++20**: 使用现代 C++ 特性

## 注意事项

1. **OpenCV 支持**: 如果没有 OpenCV 支持，工具会显示相应警告
2. **图像格式**: 某些特殊格式可能需要额外的编解码器支持
3. **性能**: 大图像处理可能需要较长时间，请耐心等待
4. **内存**: 处理超大图像时请注意系统内存使用情况

## 扩展功能

该模块设计具有良好的扩展性，可以轻松添加新的图像处理算法：

1. 在 `ProcessingMode` 枚举中添加新模式
2. 在 `ParameterPanel::updateForMode` 中添加参数界面
3. 在 `ProcessingWorker::processImage` 中实现处理逻辑
4. 在模式选择下拉框中添加选项
