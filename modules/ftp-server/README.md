# FTP 服务器工具

这是一个功能完整的 FTP 服务器工具，支持多用户管理、权限控制和实时监控。

## 功能特性

### 🚀 服务器管理
- **端口配置**: 自定义FTP服务端口（默认21）
- **根目录设置**: 配置FTP服务器根目录
- **连接数限制**: 设置最大并发连接数
- **被动模式**: 支持被动模式，配置端口范围
- **一键启停**: 便捷的服务器启动/停止控制

### 👥 用户管理
- **多用户支持**: 创建和管理多个FTP用户
- **细粒度权限**: 独立控制读取、写入、删除、创建目录权限
- **用户目录**: 为每个用户设置独立的主目录
- **用户状态**: 启用/禁用用户账户
- **用户描述**: 为用户添加备注信息

### 📊 实时监控
- **连接监控**: 实时查看活动连接列表
- **传输统计**: 监控每个连接的数据传输量
- **连接管理**: 可手动断开特定客户端连接
- **服务器状态**: 显示服务器运行状态和网络信息
- **详细日志**: 记录所有连接和操作日志

### 🔒 安全特性
- **权限控制**: 精确控制用户文件操作权限
- **路径限制**: 用户只能访问指定目录
- **连接限制**: 防止过多连接影响服务器性能
- **操作日志**: 记录所有用户操作便于审计

## 使用方法

### 1. 服务器配置

#### 基本配置
1. **端口设置**: 在"服务器配置"标签页设置FTP端口（默认21）
2. **根目录**: 选择FTP服务器的根目录
3. **服务器名称**: 设置服务器标识名称
4. **最大连接数**: 设置允许的最大并发连接数

#### 高级配置
1. **匿名访问**: 选择是否允许匿名用户访问
2. **被动模式**: 启用被动模式并设置端口范围
3. **端口范围**: 配置被动模式的端口范围（如50000-50100）

### 2. 用户管理

#### 添加用户
1. 切换到"用户管理"标签页
2. 点击"添加用户"按钮
3. 填写用户信息：
   - **用户名**: 唯一的用户标识
   - **密码**: 用户登录密码
   - **主目录**: 用户的根目录路径
   - **权限设置**: 选择用户可执行的操作
   - **描述**: 添加用户说明

#### 权限说明
- **读取文件**: 允许下载和浏览文件
- **写入文件**: 允许上传和修改文件
- **删除文件**: 允许删除文件和目录
- **创建目录**: 允许创建新目录

### 3. 启动服务器

1. 完成配置后点击"启动服务器"按钮
2. 服务器成功启动后状态变为"运行中"
3. 在"服务器状态"标签页查看服务器信息
4. 客户端可通过显示的IP地址和端口连接

### 4. 监控管理

#### 连接监控
- 在"连接监控"标签页查看活动连接
- 显示客户端IP、用户名、连接时间等信息
- 可手动断开特定连接

#### 日志查看
- 在"服务器日志"标签页查看详细操作日志
- 包含连接、认证、文件操作等记录
- 按时间顺序显示所有活动

## 技术实现

### 核心组件

1. **FTPServer**: 主界面控制器
   - 管理整体界面布局
   - 协调各个组件交互
   - 处理配置保存加载

2. **FTPServerCore**: 服务器核心
   - 基于QTcpServer实现
   - 管理客户端连接
   - 处理用户认证

3. **FTPClientConnection**: 客户端连接处理
   - 实现FTP协议命令
   - 处理文件传输
   - 管理数据连接

4. **FTPUserManager**: 用户管理组件
   - 用户增删改查
   - 权限配置界面
   - 用户状态管理

5. **FTPConnectionMonitor**: 连接监控组件
   - 实时连接状态
   - 传输统计
   - 连接管理

### 协议支持

实现了标准FTP协议的核心命令：

#### 连接和认证
- **USER**: 用户名认证
- **PASS**: 密码认证
- **QUIT**: 断开连接

#### 目录操作
- **PWD**: 显示当前目录
- **CWD**: 改变目录
- **LIST**: 列出目录内容
- **MKD**: 创建目录
- **RMD**: 删除目录

#### 文件操作
- **RETR**: 下载文件
- **STOR**: 上传文件
- **DELE**: 删除文件

#### 数据传输
- **PASV**: 被动模式
- **PORT**: 主动模式
- **TYPE**: 传输类型

### 安全实现

1. **路径验证**: 防止目录遍历攻击
2. **权限检查**: 每次操作前验证用户权限
3. **连接限制**: 防止DoS攻击
4. **错误处理**: 安全的错误信息返回

## 配置说明

### 默认配置
```cpp
FTPServerConfig config;
config.port = 21;                    // FTP端口
config.maxConnections = 10;          // 最大连接数
config.allowAnonymous = false;       // 禁止匿名访问
config.usePassiveMode = true;        // 启用被动模式
config.passivePortMin = 50000;       // 被动端口范围
config.passivePortMax = 50100;
```

### 默认用户
- **用户名**: admin
- **密码**: admin
- **权限**: 完全权限（读写删除建目录）

## 客户端连接

### 使用FTP客户端连接
```bash
# 命令行FTP客户端
ftp 192.168.1.100
> admin
> admin

# 或使用工具内置的FTP客户端
# 在工具列表中选择"FTP客户端"进行连接测试
```

### 连接参数
- **主机**: 服务器IP地址
- **端口**: 配置的FTP端口（默认21）
- **用户名**: 在用户管理中创建的用户
- **密码**: 对应的用户密码
- **模式**: 建议使用被动模式

## 故障排除

### 常见问题

1. **无法启动服务器**
   - 检查端口是否被占用
   - 确认有足够的系统权限
   - 验证根目录是否存在

2. **客户端连接失败**
   - 检查防火墙设置
   - 确认服务器正在运行
   - 验证用户名密码

3. **被动模式问题**
   - 检查被动端口范围
   - 确认防火墙允许端口范围
   - 验证客户端被动模式设置

4. **文件传输失败**
   - 检查用户权限设置
   - 确认磁盘空间充足
   - 验证目录访问权限

### 调试建议

1. 查看服务器日志了解详细错误信息
2. 使用内置FTP客户端测试连接
3. 检查用户权限配置
4. 验证网络连接和防火墙设置
